Advanced Operating System Development with xv6

  • Category: Operating System Internals
  • Languages: C, Assembly
  • Project date: March, 2023
  • Project URL: https://github.com/Rahil-Parikh/operating_system_xv6

Description:

This project centers around advancing operating system development using the xv6 kernel tailored for the RISC-V architecture. It entails the implementation of crucial components like Boot Rom and Bootloader, Process Memory Management, User-Level Thread Management, and Trap and Emulate Virtualization. Each component's implementation is segregated into distinct branches within the repository.

Objective:

The objective of this project is to provide a hands-on platform for learning and experimentation with operating system development principles utilizing the xv6 kernel. It presents a structured approach to comprehend and deploy fundamental OS functionalities, fostering a deeper comprehension of system-level programming and OS internals.

Boot ROM and Bootloader:

  • Developed a custom bootloader for the xv6 OS kernel, leveraging QEMU for system emulation.
  • Configured hardware settings to ensure optimal OS operation.
  • Implemented functionality to load the appropriate OS based on user configuration.
  • Provided the OS with system configuration information, such as the bootloader binary's starting address.

Process Memory Management in xv6:

  • Implemented on-demand paging for process memory management, addressing page faults.
  • Modified the static page allocation mechanism to accommodate dynamic page loading.
  • Wrote a page fault handler to intercept and load pages upon occurrence of page faults.
  • Enabled on-demand loading of program binary contents and heap memory utilization.
  • Implemented page swapping to disk during page faults for efficient memory usage.

Trap and Emulate Virtualization:

  • Designed a virtual machine using the "trap and emulate" approach.
  • Executed user-level instructions directly within the virtual machine, while trapping supervisor-level instructions.
  • Initialized privileged virtual machine state and tracked VM execution.
  • Redirected trapped instructions to the virtual machine monitor (VMM) for emulation.
  • Emulated privileged instructions and maintained privileged state, including critical registers.
  • Implemented emulation of physical memory protection (PMP) for enhanced security.

Conclusion:

By delving into the intricacies of advanced OS development concepts collaboratively, this project delivers a comprehensive learning journey. It equips participants with the requisite knowledge and skills to navigate intricate OS architectures and contribute innovatively to the field's advancements.
